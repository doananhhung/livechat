# Project Guidelines

## Language Standards
- **Backend**: All text, strings, error messages, and logs in the backend must be in **English only**.
- **Frontend**: All user-facing text in the frontend must follow the internationalization protocols defined in `packages/frontend/src/i18n`. Hardcoded strings are not allowed; use translation keys.

## Testing Frameworks
- **Backend**: Use **Jest** for all backend testing (Unit, Integration, E2E).
- **Frontend**: Use **Vite** (Vitest) for all frontend testing.
